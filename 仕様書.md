# AIチャットボット 仕様書

## 概要

最低限のチャット機能を備えたAIチャットボットアプリケーションをNext.jsで実装する。
ユーザはメインプロンプト、役割、スキルを入力し、OpenAIのAPIを通じてチャットを行うことができる。チャット履歴はサイドバーに表示され、過去の会話にアクセスできる。
dbは使用せず、サンプルデータを用いてチャット履歴を管理する。
まずはAPIにアクセスせず、固定の文字列を返すことで、フロントエンドのUIとUXを優先的に実装する。

## 実装範囲

- チャット画面
  - チャット機能
    - メインプロンプトの入力
    - ユーザの役割、スキルの入力
    - OpenAIのAPIへプロンプトを送信（後ほど実装）
    - OpenAIのAPIからのレスポンスを受信（後ほど実装）
    - レスポンスを表示
  - (サイドバー)チャット履歴
    - チャット履歴一覧の取得
    - チャット履歴の表示
    - チャット履歴の選択

## 使用技術

- フロントエンド・バックエンド共通
  - Next.js
  - shadcn/ui（UIコンポーネントライブラリ）
  - Tailwind CSS（スタイリング。カラーテーマ定義）
- OpenAI API（チャット機能のバックエンド）

## 実装方針

### 概要

- データのロード中は`npx shadcn-ui@latest add skeleton`を使用して各要素のスケルトンを表示
- サンプルデータを使用し、まずはフロントエンドの機能とUIを完成させる
- shadcn/uiのコンポーネントを活用してモダンで統一感のあるデザインを実現
  - shadcn/uiでコンポーネントがあれば、それを使い、なければ自作する
    - それにより、コードの可読性を確保

### 詳細

#### app/page.tsx（メインページ）

- 状態
  - ロード状態なし（静的コンテンツ）
- 実装
  - ビジネス向けのクールなランディングページを作成
  - Hero セクション、機能説明、チャット開始ボタンを配置
  - chat/page.tsx へのナビゲーションボタンを実装

#### app/chat/page.tsx（チャット開始ページ）

- 状態
  - ロード状態なし(画面なし)
- 実装
  - ユニークなchatIdを生成（UUID v4を使用）
  - 生成したchatIdでchat/[id]ページにリダイレクト

#### app/chat/[id]/page.tsx（チャット画面）

- 状態
  - `chatHistories: ChatHistorySummaryObject[]`（チャット履歴一覧）
  - `currentChat: ChatHistoryObject | null`（現在のチャット詳細）
  - `inputPrompt: InputPromptObject`（入力フォームの状態）
  - `isLoading: boolean`（ロード状態）
  - `isSidebarOpen: boolean`（サイドバーの表示状態）
- 実装
  - useEffectでマウント時にチャット履歴一覧と選択されたチャット履歴を取得
  - useStateでInputPromptObjectを管理
  - components/chat/以下のコンポーネントを組み合わせてチャット画面を構築

#### components/chat/ChatForm.tsx

- 状態
  - `formData: InputPromptObject`（フォーム入力値）
  - `isSubmitting: boolean`（送信中状態）
- 実装
  - メインプロンプト、ユーザー役割、スキルの入力フィールド
  - ChatFormUserProfile.tsxを内包
  - 送信時にAPIエンドポイントへリクエスト（モック実装）
  - カードのようなデザインでフォームを囲む
    - npx shadcn@latest add cardでカードコンポーネントを追加
  - タブでメインプロンプト/ユーザープロフィールを切り替え
    - npx shadcn@latest add tabsでタブコンポーネントを追加

#### components/chat/ChatFormUserProfile.tsx

- 状態
  - `isExpanded: boolean`（プロフィール入力欄の展開状態）
- 実装
  - 折りたたみ可能なユーザープロフィール入力セクション
  - 役割とスキルの入力フィールド

#### components/chat/ChatMessage.tsx

- 状態
  - プロパティとして受け取る`message: MessageObject`
- 実装
  - ユーザーメッセージとアシスタントメッセージの表示分け（ユーザーは右、アシスタントは左）
  - マークダウン対応でレスポンス内容をレンダリング
  - タイムスタンプの表示
  - コピー機能の実装
  - マークダウンを表示
    - 参考：[Next.js でマークダウンと MDX を使用する方法](http://nextjs.org/docs/app/guides/mdx)
      - その他、マークダウン用の使いやすいライブラリがあれば、それでも良い。

#### components/chat/ChatHistorySideBar.tsx

- 状態
  - プロパティとして受け取る`histories: ChatHistorySummaryObject[]`
  - プロパティとして受け取る`currentChatId: string | null`
- 実装
  - npx shadcn@latest add sidebarでサイドバーを追加
  - チャット履歴一覧の表示
  - 現在のチャットをハイライト
  - 新規チャット開始ボタン
  - レスポンシブ対応（モバイルではオーバーレイ）
  - ダミーのプロフィールAvatar表示

#### app/api/db/chat/route.ts

- 実装
  - sample_data/chat_history.jsonからチャット履歴一覧を読み込み
  - あえて1.0秒の遅延を追加してロード状態を確認できるようにする
  - ChatHistorySummaryObject[]形式でレスポンス
  - エラーハンドリング実装

#### app/api/db/chat/[id]/route.ts

- 実装
  - sample_data/chat_history.jsonから指定されたIDのチャット詳細を取得
  - ChatHistoryObject形式でレスポンス

- 実装
  - フロントエンドからのInputPromptObjectを受け取り
  - モック実装として固定の文字列レスポンスを返す

#### sample_data/chat_history.json

- 実装

  - ChatHistoryObject形式でメッセージ履歴を含む
  - 開発とテスト用の豊富なサンプルデータ

#### sample_data/user_profiles.json

- 実装
  - ユーザープロフィールのサンプルデータを格納

  - フォーム入力のプリセット機能で使用予定

#### lib/types.ts

## 型定義（Types.ts）

### 概要

export interfaceを使用して、各データ構造の型を定義する。

```typescript
export interface ChatHistorySummaryObject {
  chatId: string;
  title: string;
  createdAt: string;
  updatedAt: string;
}
```

### 型情報

- チャット履歴一覧（ChatHistorySummaryObject）
  - chatId: string (チャットの一意な識別子)
  - title: string (チャットのタイトル。最初のユーザーメッセージの一部を使用)
  - createdAt: string (チャットの作成時間)
  - updatedAt: string (チャットの最終更新時間)
- チャット履歴詳細（ChatHistoryObject）
  - chatId: string (メッセージの一意な識別子。時間に依存したUUID)
  - messages: Message[] (メッセージの配列)
  - createdAt: string (チャットの作成時間)
  - updatedAt: string (チャットの最終更新時間)
- メッセージ (MessageObject)
  - role: 'user' | 'assistant' (メッセージの役割)
  - content: string (メッセージの内容)
  - timestamp: string (メッセージのタイムスタンプ)
- 入力プロンプト (InputPromptObject)
  - mainPrompt: string (メインプロンプト)
  - userRole: string (ユーザの役割)
  - userSkills: string (ユーザのスキル)
